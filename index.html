🗃️ 1. O que é uma tabela?

Uma tabela é como uma planilha dentro do banco de dados. Ela guarda informações organizadas em linhas e colunas.

Colunas: representam os tipos de dados, como nome, valor, idade e descrição.
Linhas: representam os registros cadastrados, ou seja, os dados de verdade.
idnomevalor
1Arroz25.90
2Feijão8.50

Cada linha da tabela acima representa um produto diferente.

🔢 2. Tipos de dados básicos

INTEGER

Usado para números inteiros.

FLOAT

Usado para números com casas decimais.

valor FLOAT
preco FLOAT

TEXT

Usado para textos.

nome TEXT
descricao TEXT

Criando estrutura

🏗️ 3. Como criar uma tabela

Para criar uma tabela usamos o comando CREATE TABLE.

CREATE TABLE produto (
    id INTEGER,
    nome TEXT,
    valor FLOAT
);

Esse código cria uma tabela chamada produto com três colunas: id, nome e valor.

Inserindo registros

➕ 4. Como inserir dados

Para inserir dados usamos o comando INSERT INTO.

INSERT INTO produto (id, nome, valor)
VALUES (1, 'Arroz', 25.90);

INSERT INTO produto (id, nome, valor)
VALUES (2, 'Feijão', 8.50);
Consultando dados

🔍 5. Como consultar dados

Para visualizar os dados usamos o comando SELECT.

SELECT * FROM produto;

O símbolo * significa: trazer todas as colunas da tabela.

🔑 6. Chaves primárias e estrangeiras

PRIMARY KEY

A chave primária identifica cada registro de forma única. Ela não deve se repetir.

id INTEGER PRIMARY KEY

FOREIGN KEY

A chave estrangeira liga uma tabela à outra. Ela cria o relacionamento entre os dados.

FOREIGN KEY (id_vendedor)
REFERENCES vendedor(id)
Exemplo completo

🔗 7. Relacionamento 1:N

Um vendedor pode ter vários produtos. Mas cada produto pertence a apenas um vendedor.

Isso é um relacionamento 1:N, ou seja, um para muitos.

VENDEDOR
  • id PK
  • nome
1 → N
PRODUTO
  • id PK
  • nome
  • valor
  • id_vendedor FK

Criando as tabelas

CREATE TABLE vendedor (
    id INTEGER PRIMARY KEY,
    nome TEXT
);

CREATE TABLE produto (
    id INTEGER PRIMARY KEY,
    nome TEXT,
    valor FLOAT,
    id_vendedor INTEGER,
    FOREIGN KEY (id_vendedor) REFERENCES vendedor(id)
);

Inserindo dados

INSERT INTO vendedor VALUES (1, 'Carlos');
INSERT INTO vendedor VALUES (2, 'Ana');

INSERT INTO produto VALUES (1, 'Arroz', 25.90, 1);
INSERT INTO produto VALUES (2, 'Feijão', 8.50, 1);
INSERT INTO produto VALUES (3, 'Café', 18.00, 2);

Consultando

SELECT * FROM vendedor;
SELECT * FROM produto;
Cuidado!

🧨 8. Como excluir uma tabela

Para apagar uma tabela usamos o comando DROP TABLE.

DROP TABLE produto;
Atenção: esse comando apaga a tabela inteira, incluindo todos os dados.
